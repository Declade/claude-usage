<task type="auto">
  <name>Task 2: Generate Xcode project from XcodeGen spec</name>
  <files>project.yml, ClaudeUsage/ClaudeUsageApp.swift, ClaudeUsage/ContentView.swift, ClaudeUsage/Assets.xcassets/.gitkeep</files>
  <action>
    Create all source files and generate the Xcode project. Order matters: source files first, then project.yml, then xcodegen generate.

    **Step A — Create source files:**

    Create `ClaudeUsage/ClaudeUsageApp.swift`:
    ```swift
    import SwiftUI

    @main
    struct ClaudeUsageApp: App {
        var body: some Scene {
            MenuBarExtra("Claude Usage", systemImage: "chart.bar.fill") {
                ContentView()
                    .frame(width: 320)
            }
            .menuBarExtraStyle(.window)
        }
    }
    ```

    Create `ClaudeUsage/ContentView.swift`:
    ```swift
    import SwiftUI

    struct ContentView: View {
        var body: some View {
            VStack(spacing: 8) {
                Text("Claude Usage")
                    .font(.headline)
                Divider()
                Button("Quit") {
                    NSApplication.shared.terminate(nil)
                }
                .keyboardShortcut("q")
            }
            .padding()
        }
    }
    ```

    Create `ClaudeUsage/Assets.xcassets/.gitkeep` (empty file — XcodeGen needs the directory to exist before it generates the project).

    **Step B — Create project.yml:**
    ```yaml
    name: ClaudeUsage
    options:
      bundleIdPrefix: com.marcschuelke
      deploymentTarget:
        macOS: "14.0"
      xcodeVersion: "16.0"

    targets:
      ClaudeUsage:
        type: application
        platform: macOS
        deploymentTarget: "14.0"
        sources:
          - ClaudeUsage
        info:
          path: ClaudeUsage/Info.plist
          properties:
            LSUIElement: true
            CFBundleDisplayName: Claude Usage
            NSHumanReadableCopyright: ""
        entitlements:
          path: ClaudeUsage/ClaudeUsage.entitlements
          properties:
            com.apple.security.app-sandbox: false
        settings:
          base:
            PRODUCT_BUNDLE_IDENTIFIER: com.marcschuelke.claudeusage
            SWIFT_VERSION: "6.0"
            MACOSX_DEPLOYMENT_TARGET: "14.0"
            ENABLE_HARDENED_RUNTIME: YES

      ClaudeUsageTests:
        type: bundle.unit-test
        platform: macOS
        deploymentTarget: "14.0"
        sources:
          - ClaudeUsageTests
        dependencies:
          - target: ClaudeUsage
        settings:
          base:
            TEST_HOST: "$(BUILT_PRODUCTS_DIR)/ClaudeUsage.app/Contents/MacOS/ClaudeUsage"
            BUNDLE_LOADER: "$(TEST_HOST)"
    ```

    **Step C — Install XcodeGen if needed and generate:**
    ```bash
    # Install if missing
    which xcodegen || brew install xcodegen

    # Generate the Xcode project (run from project root containing project.yml)
    xcodegen generate
    ```

    **Step D — Verify the build:**
    ```bash
    xcodebuild build \
      -scheme ClaudeUsage \
      -destination 'platform=macOS' \
      -configuration Debug \
      | tail -20
    ```

    If the build fails, check:
    - Swift 6 warnings-as-errors: The App and View structs are @MainActor-isolated implicitly; no concurrent code in Phase 1 so no warnings expected
    - Missing Info.plist: XcodeGen generates it at `ClaudeUsage/Info.plist` — do not create it manually
    - Missing entitlements: XcodeGen generates it at `ClaudeUsage/ClaudeUsage.entitlements` — do not create it manually

    Do NOT add `.menuBarExtraStyle(.menu)` — use `.window` style only. See research pitfall #3.
    Do NOT add AppDelegate or NSApplicationDelegate — not needed with MenuBarExtra.
    Do NOT import AppKit directly in ClaudeUsageApp.swift — SwiftUI is sufficient.
  </action>
